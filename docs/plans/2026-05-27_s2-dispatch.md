## S2b — sync loop (md↔db, 5s tick, all subpages + dashboard)

**Goal**: every 5s, for each subpage AND `dashboard.md`, if md newer than db → reconcile (md → db); if db newer than md → render (db → md). hash-guard preserves manual edits at row level.

**Background**:
- Current state: watcher 200ms only observes hash (`marrow/watcher.py:213-225`, `_fire_sync` → `sync_file_observe`), does NOT call reconcile or render. Reconcile only runs inside `write_subpage` when `mw refresh` (or similar) is invoked.
- Existing hash-guard (`marrow/subpages.py:112-121`) runs reconcile BEFORE render, so re-rendering does not clobber manual edits.
- Row-level protection: each tracked block has `<!-- id:<table>.<id> -->` + content_hash; if stored hash ≠ cur hash, inserter preserves user body on render.

**Design**:
- New module `marrow/sync_loop.py`
- Loop runs in watcher thread, tick = 5s (constant `_SYNC_TICK_S = 5.0`, configurable via env)
- Targets:
  - Every entry in `subpages._REGISTRY`
  - `dashboard.md` (not in `_REGISTRY` — has separate `write_dashboard` path). db_mtime = max(updated_at) across affect / task / handover / monitor / alerts / milestone_candidate tables; reconcile via existing `reconcile_affect`, `reconcile_tasks`, etc.; render via `write_dashboard`
- Per-target loop body:
  1. resolve md path
  2. read md mtime (skip if file missing)
  3. read db_mtime (per-target helper `last_db_mtime()`; if no underlying table, skip)
  4. if `md_mtime > db_mtime + 1s` (epsilon to avoid jitter): call reconcile callback(s) if defined; after reconcile, re-check md mtime — if it advanced again during reconcile, skip render this tick (avoid clobber race)
  5. elif `db_mtime > md_mtime + 1s`: call `write_subpage(cfg)` or `write_dashboard(conn)`
  6. else: skip
- Loop runs in its own thread; takes watcher's existing conn or opens its own (sqlite WAL handles concurrent readers)
- Boot behaviour: run one tick immediately after `_reconcile_boot` so drift while watcher was down gets caught

**Race防御 — reconcile-then-write race**:
- If user saves md mid-reconcile (e.g. you're typing in Obsidian): re-check md mtime AFTER reconcile, BEFORE render. If md advanced → skip render this tick, next tick (5s later) will pick up new content via reconcile then render.

**Tests** (`tests/test_sync_loop.py`):
- tick fires per loop iteration (use monkey-patched clock or short tick for test)
- md newer than db → reconcile called, render not called
- db newer than md → reconcile + render both called (render via `write_subpage` / `write_dashboard`)
- equal mtimes (within 1s epsilon) → no calls
- subpage with no reconcile callback: skip md→db direction
- dashboard target: db_mtime = max across affect/task/handover/monitor/alerts/milestone tables
- race防御: mid-reconcile md write → render skipped, next tick picks up
- multiple targets: each processed independently
- shutdown signal stops the loop cleanly

**Commits**:
1. sync_loop module + helper `last_db_mtime` on each subpage cfg
2. watcher integration (start loop on boot, stop on shutdown)

## S2c — atlas subpage (schema + render + reconcile + depth-aware fs walk)

**Goal**: `atlas` subpage replaces `dir_tree.md`; manually editable markdown heading tree; auto-stub new dirs; preserve manual fields across rename.

**Schema** (new table `atlas`):
```sql
CREATE TABLE IF NOT EXISTS atlas (
  path TEXT PRIMARY KEY,           -- absolute expanded path
  note TEXT,                       -- one-line what this dir is
  write_hint TEXT,                 -- what belongs here
  naming_hint TEXT,                -- naming convention
  depth INTEGER NOT NULL DEFAULT 0,-- expand 0/1/2/... sub-levels under this row
  stale INTEGER NOT NULL DEFAULT 0,-- 1 if fs walk no longer finds this path
  updated_at TEXT NOT NULL
);
```

Migration: `storage._migrate_to_vNN()` (next available version), `init_db` runs it.

**Render layout** — markdown heading tree, file = `~/.config/marrow/db-pages/atlas.md`:

```
## ~/cc-lab/

### marrow/
- note: SQLite-backed memory system
- write: docs/plans/, docs/notes/
- naming: mw- prefix outside repo, snake_case in repo
- depth: 1

### external/
- note: down 下来的 OSS, 部分魔改
- write: 不在这写新代码
- depth: 0

## ~/.claude/

### rules/
- depth: 0

### skills/
- depth: 0
```

Rules:
- `##` per monitored root, root displayed with `~/` shorthand, db stores absolute
- `###` for first-level dir under each root
- `####`, `#####`, `######` for deeper levels (max 6 = h6, no deeper renders)
- Heading body = dir name with trailing `/`
- `(stale)` suffix on heading when `stale=1`
- Bullet fields: `- note:`, `- write:`, `- naming:`, `- depth:` — omit entirely if value empty
- `depth` always rendered (even when 0) so user can edit it inline

**Reconcile** (`reconcile_atlas(conn, md_path)`):
- Parse markdown headings + bullets
- Build map `{path: {note, write, naming, depth}}`
- Upsert into `atlas` table
- Paths in db not in current md → DELETE (user explicitly removed)
- Wire via `_REGISTRY` registration with `reconcile=reconcile_atlas`

**Spec** (`build_atlas_spec(folder: str) -> InserterSpec` in `marrow/subpage_specs.py`):
- Mirror existing `build_projects_index_spec` / `build_study_index_spec` pattern
- `key="atlas"`, `path` = absolute md file path
- `fetch` queries atlas table grouped by root (top-level dir under `~/`)
- `render_row` produces the heading + bullet block per row
- `render_section_header` produces `## ~/<root>/`
- `section_of` maps row to its root path

**Depth-aware fs walk** (new helper `atlas_sweep_fs(conn)` or similar):
- For each atlas row with `depth > 0`:
  - `os.walk(path, ...)` to depth `depth`
  - For each subdir found:

    - If fs no longer finds it → set `stale=1` (do NOT delete, preserves manual fields)
- Respect `EXCLUDE_DIRS_TREE` from drift_sweep (skip `__pycache__`, `node_modules`, `.git`, etc.)
