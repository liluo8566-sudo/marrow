# Marrow MAP · 简易版
> 2026-05-30
> 第一次看、没记忆,也能不翻代码就懂每块在干嘛、怎么实现。功能/初衷/机制三行,不报函数名。

```
                  CLI / 微信 / 桌面端   ← 你在这儿聊天
                        │   ▲
          对话结束 写入 │   │ 每句话 召回
                        ▼     │
        ┌─────────────────────────────────────-┐

        └─────────────────────────────────────┘
                        │
            周边：日记 · 日计划 · 铁锅 · 钱包(占位)
```

## 入库 · 把对话变成记忆

### session 捕获 + 抽取
- 功能：每次对话结束,把这次聊天存进记忆库,再后台让 sonnet 提炼出 任务 / 交接 / 情绪 / 日志 四样东西。
- 初衷：hook 一结束就会被系统杀掉,当场跑不了慢的 LLM——所以先把对话快速存档,再甩一个脱离 cc 的后台进程慢慢抽。
- 机制：对话结束 → 清掉工具调用和系统噪音、存进 events 表 → 拉起独立后台进程,跑两次 sonnet:第一次出「任务+交接」,第二次出「情绪+日志」,各自写进对应的表。

### affect 情绪
- 功能：给每段对话打情绪分(valence 正负 / arousal 强度 / 重要度),渲染成 dashboard 顶部的心情背景(当前 + 24h + 7d 趋势)。
- 初衷：召回只浮现「发生了什么事」,不浮现「当时什么心情」,情绪得单独抽一层。
- 机制：后台 sonnet 把一次会话切成几段情绪 episode,每段打 V/A/重要度 + 一句描述,写进 affect 表;主页读 最近一批 + 24h + 7d,算加权平均,落进 9 宫格(情绪带 × 强度带)翻成两个字的主调(如「温暖/愉悦」),渲成顶部背景。哪天有对话却没情绪记录,开窗时报一条警告。

### entities 实体
- 功能：从每天的对话里抽出 人 / 偏好 / 地点(妈妈、最爱的颜色、Bendigo…)存成档案;你提到谁,召回时强制把相关事件顶上来。
- 初衷：2 字短中文名、中英混的昵称,普通搜索会漏,得专门留一层按名字反查。
- 机制：每天聚合当天对话跑一次抽取 → 新实体先和已有的查重(名字精确比 + 语义相似比),撞了就并入别名、不新增 → 你每提一次某实体,它的「被提次数」+1;召回时只要 query 里出现它的名字,就把它关联的事件抢先塞进结果。

### memes & milestones 梗 & 里程碑
- 功能：抽出反复出现的梗(秃鹫、赎身费…)和一次性的人生大事(里程碑),分别存档。
- 初衷：新梗可能和已有的实体名或里程碑撞概念,得跨三张表查重防污染;里程碑还要防「删了又被重新抽出来」。
- 机制：每天一次抽取 → 梗要求 7 天内至少 3 天出现过才收(滤掉一次性词) → 跨 梗/里程碑/实体 三张表做字面 + 语义查重,被拒的记进黑名单下次直接跳过 → 里程碑额外查「墓碑」,删过的不复活。

## 记忆核心 · 存与取

### embedding 向量化
- 功能：把六类内容(对话/梗/实体/里程碑/日记/任务)转成向量,供语义搜索。
- 初衷：本地不上云(硬约束),模型锁死 bge-m3——换模型要重算全部,所以记下「谁 embed 的」以便换了能报警。
- 机制：常驻进程里跑一个本地模型(只加载一次);每张表没向量的行批量补算,写进各自的向量表;漏算的下次自动补上。

### recall 召回
- 功能：你问一句,从所有记忆表里融合搜出最相关的几张卡片。
- 初衷：单一搜法都不够——关键词搜不到语义近的,向量搜不到生僻名字;得多路融合再统一排序,还要防某一类霸占全部名额。
- 机制：同时跑 关键词 + 向量 两路搜对话,再各路扫 梗/里程碑/实体/日记/任务 → 四个信号(向量近度 + 关键词 + 新鲜度 + 情绪权重)加权打分排序 → 冷门旧记忆(重要度低且 >90 天)默认不出,除非关键词正好命中就当场复活 → 每类留名额、按字数截断。

## 展示 · 主页 + 子页面

### dashboard 主页 — 双向 ✓ (唯一真·完整双向)
- 功能：用户主入口——alert / task / affect / 里程碑候选 / 所有子页面链接,全汇在 dashboard.md 顶部一区。
- 初衷：多入口多 md 难管,all-in-one 一页看全、美观省心。
- 机制：SessionEnd 渲染各区块(db→md);刷新前先把你在 md 上的 勾选/编辑/投票 reconcile 回库,再用库重渲染,手改过的块跳过不覆盖 → 原子写盘。

> **子页面同步:设计 vs 实际(MAP 的真相)**
> 设计:**除 study / project 外,所有子页面都该双向同步**(你改 md 要进库)。
> 实际:只有 `milestone` / `atlas` 接了 reconcile 回库(加主页 = 3 个),其余 **6 个 half-done**——你改了不消失、但不进库。**这是欠的债,不是已完成。**

### milestone 里程碑 — 双向 ✓
- 功能：人生里程碑存档,分 Us / Me,pinned 的才显示。
- 初衷：一次性大事永久留底,可手动增删改。
- 机制：抽取 conf≥0.85 入 milestones 表 → 渲成 md;你在 md 投票删 / 改标题 → reconcile 回库。

### atlas 目录地图 — 双向 ✓ (子页面里机制最重,还自带独立 MCP tool)
- 功能：把 marrow 管的文件树渲成 atlas.md,每个目录带说明 + 命名提示;另挂 `atlas_lookup` MCP tool 供随时查。
- 初衷：跨目录文件多,要一张可查可维护的地图,写新文件时给命名提示。
- 机制：每次刷新先补根 + 扫文件系统(新目录 stub 进库、消失且无手填字段的删)→ 渲成 md;你改 md 的说明 / 层级 → reconcile 回库;文件改名时迁移对应 atlas 行。

### diary 日记 — half-done ✗
- 功能：每天根据当天所有 session 的聊天自动生成日记。
- 初衷：懒得手写,记录生活点滴。
- 机制：每会话 SessionEnd 先产出 digest 存库 → 每天定时(plist)跑 sonnet 把当天 digest 聚合成日记、按日期重写 diary 表 → 渲成 md。**你改 md 不回库(没接 reconcile)= 欠双向。**

### memes 梗 — half-done ✗
- 功能：梗存档,6 个 tag(paw / fact / meme / event / news / others),分 Personal(paw/fact) / Public(其余)。
- 初衷：反复出现的梗值得留,但要跨表查重防污染。
- 机制：daily 抽取入 memes 表(代码写了,实际录入是否跑通**待验**)→ 渲成 md。**你改 md 不回库 = 欠双向**;退休见 aging。

### profile 个人档案 — half-done ✗
- 功能：个人档案,源自 entities 表,按 人 / 偏好 / 地点 分组。
- 初衷：把抽出的实体档案一页看全。
- 机制：daily 抽 entities → 渲成 md。**没接 reconcile 不回库 = 欠双向**;entities 目前无自动退休。

### stickers / wallet / goose — half-done ✗ / 占位
- stickers：贴纸页,源 stickers 表,渲染→md,无回写(欠双向)。
- wallet：钱包,连表都没,纯占位空块。
- goose：铁锅每日最佳吐槽,源 goose_bites 表,渲染→md,无回写(欠双向);aging 清 7 天前的鹅日志 md。

### cheatsheet 速查表 — 磁盘为源,本就不回库
- 功能：扫 skills / hooks / 别名 渲成一页。
- 机制：read_only,磁盘是源、每次全覆盖;设计上就不回库,不算 half-done。

### study / projects — 任务派生,设计上就不双向
- 功能：学习按单元、项目按项各一页,源都是 tasks 表。
- 机制：index 页块级渲染、project 子页 read_only;退休跟 tasks(30 天没提→归档)。**这两个设计就不要双向,不算欠债。**

### handover 交接 — 独立文档,不是子页面、也不是主页的一部分
- 功能：跨窗口交接——进行中的事 / 任务 / 你的备注,写进独立的 handover.md,下个窗开窗自动读。
- 和 dashboard 的关系：俩都是 SessionEnd 写的 md,但**两个独立文件、各自机制**——主页是你日常看的,handover 是换窗口才用的;handover 不在子页面 registry 里。
- 机制：SessionEnd sonnet 对「进行中」段出增量指令(关闭/更新/保留/新增)→ 加锁、按编号改条目、完成超 24h 滚出、你手删的行不再被写回 → 原子写盘。

### md↔db 同步引擎 — 底层,撑上面所有页
- 功能：让 md 和库保持同步的底层机制(不是某一页)。
- 机制：watcher 实时监听(200ms 防抖)记每个块的 hash + 5 秒轮询比对 md/库谁新决定方向;db→md 用块级 upsert 渲染(保留你手加的块);md→db **只有接了 reconcile 的页才有**(主页 / milestone / atlas),其余页缺这一环 = half-done;文件改名走 drift_sweep 扫所有引用。

## 底座 Infra

### daemon / MCP 常驻服务
- 功能：常驻后台,给 CLI 和微信提供三个工具:召回、目录查询、补算向量。
- 初衷：hook 和 CLI 都是一次性的,只有常驻进程能实时响应召回,还能把大段记忆绕开 hook 的 1 万字上限。
- 机制：一个常驻 MCP 服务,开机初始化库后挂起等调用。

### llm 供应商抽象
- 功能：所有内部 LLM 调用走一条统一通道(默认 → 兜底 → 应急 三级)。
- 初衷：不想每个调用方都得懂 模型怎么选/超时怎么杀/被拒怎么判;还要隔离掉你的人设、工具、输出风格,别泄漏进这些后台调用。
- 机制：按「档位」选模型,依次试三级链路、各重试一次,超时杀掉整个进程组,识别拒答和报错,记下每次花销。

### config · 备份 · 原子写
- 功能：配置加载、每天数据库备份、所有 md 的原子写。
- 初衷：备份是「库永不丢」的安全网(本地 + iCloud 异地);原子写防写一半崩了留下坏文件。
- 机制：默认配置 + 你的覆盖合并出所有路径;每天导出库快照 → 原子替换到本地 → 再拷一份到异地,异地失败只报警不中断,超量旧备份自动删;所有渲染都先写临时文件再原子替换。

## 周边 Addon

### daily 日记 / 日计划
- 功能：每天早上自动写**昨天**的日记 + 从昨天对话抽 实体/梗/里程碑;日计划是早晚交互式排计划的 skill。
- 机制：mw-daily-routine.plist 每天 **07:00** 跑 → 一次 sonnet 出候选(实体/梗/里程碑)、一次出日记、加文件锁防并发;另有 mw-daily-catchup.plist **19:00** 补最近 7 天缺的日记/鹅句(最多 3 天)。

### buddy 铁锅
- 功能：状态栏那只爱吐槽的鹅,每轮结尾留一句;每天从它的吐槽里挑一句最佳存档。
- 机制：鹅的吐槽写进一个 md 日志,每天一条流水线挑当天最佳一句,渲进「鹅言鹅语」子页。

### wallet 钱包
- 功能：预算 / 花销页(占位,基本空)。
- 机制：暂时只是个挂载点,渲染空块。

## 触发器 · 什么时候自动跑

**Hook — 对话时实时**
- SessionEnd：抽情绪+日志、更新 handover、勾任务、补向量、重渲主页
- SessionStart：注入 任务/警告/情绪/交接;补跑死掉没抽完的 session(≤2 个)
- UserPromptSubmit：召回注入;`mm-` 跳过 / `mm+` 强制重跑
- PreToolUse：Write 或 mv/cp 时给 atlas 命名提示

**定时 — launchd plist**
- 03:00 备份数据库 (mw-db-backup)
- 06:01 重渲主页 (mw-dashboard-tick)
- 06:30 选当天最佳鹅句 (mw-goose-bites)
- 07:00 写昨天日记 + 抽 实体/梗/里程碑 (mw-daily-routine)
- 19:00 补漏日记/鹅句,最多 3 天 (mw-daily-catchup)
- 周日 12:00 退休清理 (mw-aging)
- 常驻 watcher 实时盯文件改动 (mw-watcher,KeepAlive)

**轮询 — watcher 进程内**
- 每 5 秒：md↔db 同步(谁新同步谁)
- 每 60 秒：atlas 扫文件系统找新目录 / 标记消失的

**按需 — MCP / CLI**
- MCP：`recall` 召回 · `atlas_lookup` 查目录 · `embed_pending` 补向量
- CLI：`mw refresh` 重渲 · `mw handover` · `mw sessionend rerun` · `mw drift` 改名扫描

## 横切机制

### aging 谁多久退休 (mw-aging.plist 周日 12:00)
- 梗：没置顶且 90 天没提 → 删
- 任务：30 天没在对话里出现 → 归档
- 里程碑提醒：加进来满 7 天没确认 → 自动标记已确认(不删)
- 鹅日志文件：超 7 天的日块 → 清掉,文件空了删
- md 索引墓碑：删除满 30 天 → 清理

### catchup 漏了怎么补
- session 抽取漏了：开窗查最近 24h,进程死了又没抽完的 → 重新补跑(每次最多 2 个)
- 日记漏了：每天扫最近 7 天有对话没日记的 → 补写(最多 3 天)
- 情绪漏了：开窗查最近 7 天有对话没情绪的 → 报警提示(只报不自动补)
- 冷门记忆复活：搜索时关键词正好命中冷门旧记忆 → 当场复活
- 向量孤儿清理：补向量前先清掉日记重写留下的失效向量
