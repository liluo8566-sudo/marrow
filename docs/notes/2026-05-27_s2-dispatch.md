## S2b ‚Äî sync loop (md‚Üîdb, 10s tick, all subpages)

**Goal**: every 10s, for each subpage, if md newer than db ‚Üí reconcile (md ‚Üí db); if db newer than md ‚Üí render (db ‚Üí md). hash-guard preserves manual edits.

**Background**:
- Current state: watcher 200ms only observes hash (`marrow/watcher.py:213-225`, `_fire_sync` ‚Üí `sync_file_observe`), does NOT call reconcile or render. Reconcile only runs inside `write_subpage` when `mw refresh` (or similar) is invoked.
- Existing hash-guard (`marrow/subpages.py:112-121`) runs reconcile BEFORE render, so re-rendering does not clobber manual edits.

**Design**:
- New module `marrow/sync_loop.py` or extend `marrow/watcher.py`
- Loop runs in watcher thread, tick = 10s (constant `_SYNC_TICK_S = 10.0`, configurable via env or config.toml)
- For each registered subpage in `subpages._REGISTRY`:
  1. resolve subpage md path
  2. read md mtime (skip if file missing)
  3. read db "last touched" for this subpage's underlying table(s) ‚Äî add a per-subpage `last_db_mtime()` helper that returns `max(updated_at)` across relevant rows; if subpage has no table, skip
  4. if `md_mtime > db_mtime`: call subpage `reconcile` callback if defined, then no render (md is source); if no reconcile defined, skip
  5. else if `db_mtime > md_mtime`: call `subpages.write_subpage(cfg, conn)` (already does reconcile then render)
  6. else: skip
- Periodic loop must not block watcher's md_index sync; run in its own thread or asyncio task
- Boot behaviour: on watcher startup, run one tick immediately after `_reconcile_boot` so any drift while watcher was down gets caught

**Cross-cutting concern ‚Äî race with user editing**:
- The 200ms md_index hash observer is the canonical "user just edited" signal. Sync loop should respect that ‚Äî when `_fire_sync` updates the md_index hash, sync loop's next tick will see `md_mtime > db_mtime` and trigger reconcile, not render. hash-guard still protects body.

**Tests** (`tests/test_sync_loop.py`):
- 10s tick fires per loop iteration (use monkey-patched clock or short tick for test)
- md newer than db ‚Üí reconcile called, render not called
- db newer than md ‚Üí reconcile + render both called (render via `write_subpage`)
- equal mtimes ‚Üí no calls
- subpage with no reconcile callback: skip md‚Üídb direction
- multiple subpages: each processed independently
- shutdown signal stops the loop cleanly

**Commits**:
1. sync_loop module + helper `last_db_mtime` on each subpage cfg
2. watcher integration (start loop on boot, stop on shutdown)
3. tests

**Done**:
- `pytest -q` exit 0

- manual test: edit a subpage md in Obsidian, wait 10s, check db row updated_at advances

## S2c ‚Äî atlas subpage (schema + render + reconcile + depth-aware fs walk)

**Goal**: `atlas` subpage replaces `dir_tree.md`; manually editable markdown heading tree; auto-stub new dirs; preserve manual fields across rename.

**Schema** (new table `atlas`):
```sql
CREATE TABLE IF NOT EXISTS atlas (
  path TEXT PRIMARY KEY,           -- absolute expanded path
  note TEXT,                       -- one-line what this dir is
  write_hint TEXT,                 -- what belongs here
  naming_hint TEXT,                -- naming convention
  depth INTEGER NOT NULL DEFAULT 0,-- expand 0/1/2/... sub-levels under this row
  stale INTEGER NOT NULL DEFAULT 0,-- 1 if fs walk no longer finds this path
  updated_at TEXT NOT NULL
);
```

Migration: `storage._migrate_to_vNN()` (next available version), `init_db` runs it.

Rules:
- `##` per monitored root, root displayed with `~/` shorthand, db stores absolute
- `###` for first-level dir under each root
- `####`, `#####`, `######` for deeper levels (max 6 = h6, no deeper renders)
- Heading body = dir name with trailing `/`
- `(stale)` suffix on heading when `stale=1`
- Bullet fields: `- note:`, `- write:`, `- naming:`, `- depth:` ‚Äî omit entirely if value empty
- `depth` always rendered (even when 0) so user can edit it inline

**Reconcile** (`reconcile_atlas(conn, md_path)`):
- Parse markdown headings + bullets
- Build map `{path: {note, write, naming, depth}}`
- Upsert into `atlas` table
- Paths in db not in current md ‚Üí DELETE (user explicitly removed)
- Wire via `_REGISTRY` registration with `reconcile=reconcile_atlas`

**Spec** (`build_atlas_spec(folder: str) -> InserterSpec` in `marrow/subpage_specs.py`):
- Mirror existing `build_projects_index_spec` / `build_study_index_spec` pattern
- `key="atlas"`, `path` = absolute md file path
- `fetch` queries atlas table grouped by root (top-level dir under `~/`)
- `render_row` produces the heading + bullet block per row
- `render_section_header` produces `## ~/<root>/`
- `section_of` maps row to its root path

**Depth-aware fs walk** (new helper `atlas_sweep_fs(conn)` or similar):
- For each atlas row with `depth > 0`:
  - `os.walk(path, ...)` to depth `depth`
  - For each subdir found:
    - If not in atlas table ‚Üí upsert stub row (path=child, depth=0, note='', stale=0)
    - If in atlas and was stale=1 ‚Üí clear stale to 0
  - For each existing atlas row under this path:
    - If fs no longer finds it ‚Üí set `stale=1` (do NOT delete, preserves manual fields)
- Respect `EXCLUDE_DIRS_TREE` from drift_sweep (skip `__pycache__`, `node_modules`, `.git`, etc.)
- Respect `~/.claude` whitelist (only recurse into whitelisted top-level dirs under `~/.claude`)

**Triggers** (call atlas_sweep_fs):
- `mw refresh atlas` (already routed via `_REGISTRY`)
- DriftWatcher rename event (S2a) ‚Äî when a watched path is renamed under an atlas root, trigger sweep to re-key
- Sync loop (S2b) ‚Äî sync loop tick already covers md/db direction; fs walk is independent, run on watcher boot + every N=60s tick (slower than md sync)
